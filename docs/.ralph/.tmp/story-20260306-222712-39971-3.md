### US-001: Store API access
Status: in_progress
Depends on: None

Description:
As a Shiori user, I want to configure the app with the Shiori API base URL and API key so that shared links can be saved without re-entering credentials.

Acceptance Criteria:
- [ ] `app/src/main/kotlin/dev/shiori/android/MainActivity.kt` and `app/src/main/res/layout/activity_main.xml` expose fields for server URL and API key before any share-save flow can continue.
- [ ] Server URL validation accepts the documented production base URL `https://www.shiori.sh` and current Android emulator or local development hosts, while invalid URLs keep the user on the access screen.
- [ ] API keys are normalized, stored with encrypted preferences, and can be cleared without wiping the saved server URL.
- [ ] A connection check validates stored credentials against the Shiori API before the browser or save screen becomes the default path for shared URLs.
