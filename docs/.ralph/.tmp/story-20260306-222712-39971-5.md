### US-003: Handle shared URLs from Android
Status: in_progress
Depends on: US-001

Description:
As an Android user, I want to send a URL to the app from the system share sheet or a browsable link intent so that I can save it to Shiori in one step.

Acceptance Criteria:
- [ ] `app/src/main/AndroidManifest.xml` registers the app for `Intent.ACTION_SEND` with `text/plain` and `Intent.ACTION_VIEW` for `http` and `https` URLs.
- [ ] Incoming intent parsing extracts a single supported HTTP or HTTPS URL and rejects unsupported text payloads without attempting a network request.
- [ ] `Intent.ACTION_SEND_MULTIPLE` is out of scope for v1 and shows unsupported feedback instead of batching POST requests.
- [ ] When valid API access is missing or has unsaved changes, the app retains the pending shared URL until the user finishes access setup.
- [ ] Unsupported shares show a clear status message instead of crashing or leaving the app in an undefined state.
- [ ] Cold-launch and `onNewIntent` handling avoid duplicate processing of the same shared URL.
