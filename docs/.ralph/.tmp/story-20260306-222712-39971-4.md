### US-002: Align POST /api/links client
Status: in_progress
Depends on: None

Description:
As the app, I want a network client that matches `docs/api-reference.md` for link creation so that shared URLs are submitted with the documented request and response contract.

Acceptance Criteria:
- [ ] `core-network` sends `POST /api/links` with bearer auth and a JSON body limited to `url`, optional `title`, and optional `read`.
- [ ] The create-link response model accepts `success`, `linkId`, and optional `duplicate`, and treats `linkId` as an opaque string instead of requiring a hydrated `LinkResponse`.
- [ ] The save flow does not assume metadata extraction is complete when the POST returns; any saved-link details shown in the UI come from a follow-up list refresh.
- [ ] HTTP 400, 401, 404, 409, 429, server, and network failures are mapped into domain errors that the app can surface to users.
- [ ] `core-network/src/test/kotlin/dev/shiori/android/corenetwork/ShioriApiClientTest.kt` verifies the request path, auth header, request body, duplicate handling, and error mapping for create-link.
