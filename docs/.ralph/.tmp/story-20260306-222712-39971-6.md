### US-004: Auto-save shared URLs and show result
Status: in_progress
Depends on: US-002, US-003

Description:
As a Shiori user, I want a received shared URL to be saved immediately and to see whether it was added or already existed so that I can trust the import flow.

Acceptance Criteria:
- [ ] When a supported shared URL arrives and saved API access is valid, the app routes into the main screen and reuses the add-link save workflow instead of maintaining a separate POST path.
- [ ] The auto-save request normalizes the shared URL, sends `read=false` by default, and leaves `title` empty unless a supported source value is explicitly available.
- [ ] Success, duplicate, unauthorized, conflict, rate-limited, and network outcomes each render a user-visible status message.
- [ ] After a successful or duplicate save, the app refreshes the relevant list state and lands on the destination that contains the saved link.
- [ ] `app/src/androidTest/kotlin/dev/shiori/android/MainActivityTest.kt` covers cold-launch share import, deep-link import, missing access, and unsupported shared content.
